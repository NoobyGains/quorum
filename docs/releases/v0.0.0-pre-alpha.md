# Quorum v0.0.0-pre-alpha

> Design lock-in. No runtime yet.

## What this is

Quorum is a design-locked proposal for coordinating peer AI coding agents through a typed, git-backed artifact system instead of chat. The core idea is simple: if Claude and Codex are working the same repo, coordination should happen through structured plans, reviews, handoffs, disagreements, and decisions — not freeform message history.

This tag ships the thesis, architecture, protocols, fleet-mode model, and cost/throttling model for that system. It does not ship the runtime those docs describe.

## What's shipped in this tag

- ✅ Repo scaffold and core project docs
- ✅ Design docs for architecture, protocols, fleet mode, and cost/throttling
- ✅ Roadmap with milestones M0 through M5
- ✅ GitHub issue templates, PR template, and labels for bug / feature / design-spike intake
- ✅ CI scaffold with placeholder jobs and a multi-OS / multi-Node matrix

## What's explicitly NOT shipped

- ❌ No runnable coordination system: no artifact store, SQLite index, watchdog daemon, MCP server, CLI, merge gate, dashboard, or worker fleet
- ❌ No live Claude/Codex protocol execution, no enforced cross-vendor review, and no real fleet-mode operation
- ❌ No completed implementation milestone; M0 through M5 are still ahead

## Roadmap snapshot

- **M0** — foundation and bootstrap: repo setup, CI, docs, early stubs
- **M1** — core protocol and first real Claude/Codex end-to-end flow
- **M2** — structured disagreement plus human-broker escalation
- **M3** — calibration, observability, dashboard, and cost visibility
- **M4** — specialization and cost-aware routing
- **M5** — fleet campaigns, reviewer pools, merge conductor, kill-switches

## Why this design

- Treats coordination as typed artifacts, not chat transcripts
- Uses git refs as the coordination substrate — history and auditability come for free
- Makes intent-before-action and merge-requires-review protocol rules, not etiquette
- Bounds disagreement, formalizes handoff, and plans for calibration over time

## Who this is for

Anyone already running Claude and Codex in parallel on a shared repo and feeling the failure modes of ad hoc coordination: duplicate work, branch collisions, silent scope drift, unclear review ownership.

## Known limits

- Pre-alpha: don't run anything important through it yet
- This tag is docs, structure, and GitHub scaffolding; implementation starts at M0
- The simulations and cost/throttling numbers are design estimates, not measured runtime results
- Cross-machine support and broader model support are later or explicitly unscoped
